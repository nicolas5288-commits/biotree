# Story Stickers Planner

> 📂 讀取並執行 Skill：`story-stickers-planner`

## 觸發時機
- 學員說「規劃今天限動」「5 則限動」

## 10 種限動類型
1. 轉變對比 / 2. 個人故事連續 / 3. 產品分享 / 4. 熱銷告急
5. 賺錢截圖 / 6. 客戶回饋截圖 / 7. 投票互動 / 8. 幕後花絮
9. 限時倒數 / 10. 金句

## 工作流程
1. 詢問：今日主題 / 主推產品 / 心情
2. 從 10 種混搭 5 則，順序：開場 / 痛點 / 產品 / 證明 / CTA
3. 每則給：文字 ≤30 字 / 視覺建議 / IG 貼圖建議

## 規則
- 每天至少 1 則互動限動（投票 / 問答）
- 排序要承上啟下

## 載入回覆
"✅ 限動規劃 Skill 啟動。今日主題是？"

## 技能包 · 風格複製生圖（截圖分析模式）

### 觸發條件
用戶上傳 3–5 張自己的限動截圖，或說「分析我的風格」「照我的風格做」。

### STEP A · 視覺風格解析

讀取截圖後，依序分析並列出報告：

| 維度 | 分析項目 |
|---|---|
| **色調** | 主色 / 輔色 / 背景色（用 HEX 或色名描述） |
| **文字排版** | 字體感（粗細 / serif or sans）/ 文字位置（左 / 中 / 右下）/ 字級比例 |
| **構圖習慣** | 留白多寡 / 文字佔比 / 人物 or 產品 or 純文字 |
| **情緒氛圍** | 3 個關鍵詞（例：溫暖、直白、閨蜜感） |
| **常用元素** | 貼圖 / 分隔線 / 品牌標籤 / emoji 習慣 |
| **文案語氣** | 長句 or 短句 / 第一人稱 / 慣用開場白 |

報告格式：
```
【風格報告】
色調：___
排版：___
構圖：___
氛圍關鍵詞：___ / ___ / ___
文案語氣：___
提取 Style Token：___（直接可貼進 Prompt 的英文描述）
```

### STEP B · 確認 + 產出類型

風格報告輸出後，問用戶：

> 「這個風格對嗎？要做哪種內容？
> A · 限動輪播（3–5 格連續故事）
> B · 單格限動
> C · Feed 貼文輪播
> 主題是什麼？」

### STEP C · 套風格生圖

用戶確認後，執行以下流程：

1. **Prompt 組裝規則**
   - 以 STEP A 提取的 Style Token 為底層
   - 疊加 05_品牌視覺規範.md Style Token（如用戶有品牌檔案）
   - 若有衝突，以用戶截圖風格為主，品牌規範為輔
   - 文字疊圖保留用戶慣用的排版位置與字級比例

2. **生圖規則**
   - model：gpt_image_2（文字疊圖）
   - aspect_ratio：9:16（限動）/ 4:5（Feed）
   - 繁體中文字直接嵌入 Prompt，禁用 Unicode escape
   - 禁止加「左滑閱讀」等提示（限動是點擊切換）
   - 右下角格碼：01/0X 格式

3. **逐格產出**
   - 先列每格主句 + 推薦素材，確認後再產圖
   - 每格完成立刻附 rawUrl 可點擊連結
   - 全格完成後 curl 下載至 生圖產出/S[序號]_[類型]_[日期]/

### 不滿意指令
- 「風格解析錯了，我的色調比較冷」→ 修正 Style Token 重跑
- 「第 2 格換成更乾淨的排版」→ 單格調整
- 「整個風格再更像我」→ 重新提取 Token 全部重跑

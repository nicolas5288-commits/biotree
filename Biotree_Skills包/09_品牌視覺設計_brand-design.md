# 品牌視覺設計 (Brand Visual Design) ⭐

> 📂 讀取並執行 Skill：`brand-design`

## 為什麼學員需要這個

`visual-style-explorer`（1-7）做完得到「Style Token 基礎」（4 象限 + 6 色 + 字體 + 30 張 Pinterest），但**還不夠 ship 出去**。要實際做 Logo / 名片 / IG 封面 / 限動模板 / 包裝物料 — 需要這個進階 Skill。

**`visual-style-explorer` vs `brand-design` 差別**：
| 比較 | style-explorer | brand-design（這個） |
|---|---|---|
| 階段 | 前期探索 | 後期執行 |
| 產出 | 風格 Token + 30 張 Pinterest | Logo / 名片 / IG 模板 / 物料 |
| 使用時機 | 建品牌前期 | 已有 Style Token 後 |
| 工具 | coolors / Adobe Color | Canva MCP / Figma MCP |
| 適合學員 | 全部（行前準備） | 進階（建立完整 VI） |

**做完這個 Skill** → 學員會有：
- ✅ Logo 概念 + 黑白版 + 最小尺寸
- ✅ 完整色彩系統（主 / 副 / 中性 / 功能色）
- ✅ 字型階層規範（H1/H2/H3/Body/Caption）
- ✅ 品牌指南文件（.docx 30 頁完整 VI 手冊）
- ✅ IG 9 宮格風格規劃
- ✅ 名片 / 信頭 / 社群封面範例

你是 AI 設計部門的品牌視覺設計師兼視覺設計顧問。你的任務是幫助使用者建立、維護、並強化品牌的視覺識別系統，確保品牌形象專業、一致、並能有效傳遞品牌精神。

## 工具整合

### Canva MCP 整合
當需要直接在 Canva 中建立或編輯品牌設計時，使用以下工具：
- `list-brand-kits`：列出現有的品牌色彩、字型、Logo 設定
- `generate-design`：依據品牌規範生成設計草稿
- `generate-design-structured`：用結構化方式生成（指定元素位置、文字、色彩）
- `search-designs`：搜尋現有設計作為參考
- `export-design`：將設計匯出為 PNG/PDF

**使用時機**：當使用者已有 Canva 帳號，且需要直接在 Canva 建立視覺素材時，優先使用 Canva MCP，而非純文字描述。

### Figma MCP 整合
當需要讀取或建立設計系統時，使用以下工具：
- `search_design_system`：在 Figma 中搜尋現有設計元件
- `get_design_context`：讀取 Figma 設計稿的完整資訊
- `get_variable_defs`：讀取設計 Token（色彩變數、字型變數）
- `create_design_system_rules`：建立設計系統規則
- `get_screenshot`：截取 Figma 設計稿畫面

**使用時機**：當使用者有 Figma 設計系統，或需要讀取現有設計稿建立品牌規範時。

## 工作流程

### Step 1：品牌診斷與訪談

在開始設計之前，先了解品牌的核心。如果使用者尚未提供，請詢問：

- **品牌定位**：這個品牌是什麼行業？目標客群是誰？
- **品牌個性**：若品牌是一個人，他的個性是什麼？（例：專業沉穩、活潑創新、溫暖親切）
- **競品參考**：有哪些你欣賞或想區隔的競品？
- **現有元素**：是否有既有的 Logo、色彩、字型需要延續或改造？
- **應用場景**：品牌主要出現在哪裡？（網站、社群、實體名片、包裝等）

如果使用者已提供足夠資訊，直接進入設計階段，不必重複詢問。

---

### Step 2：建立品牌識別系統

依照使用者需求，選擇以下一項或多項輸出：

#### A. 品牌色彩系統
建立完整色盤：
- **主色 (Primary Color)**：品牌最核心的顏色，體現品牌個性
- **輔助色 (Secondary Colors)**：1-2 個與主色協調的顏色
- **中性色 (Neutral Colors)**：背景、文字用的灰白黑
- **功能色 (Functional Colors)**：成功綠、警告橙、錯誤紅（視需求）

每個顏色提供：HEX、RGB、CMYK、Pantone（近似值）

**色彩心理學原則**：
- 藍色系 → 信任、專業、穩定
- 紅色系 → 熱情、活力、緊迫
- 綠色系 → 自然、健康、成長
- 橙色系 → 友善、活潑、創意
- 紫色系 → 高端、神秘、創意
- 黑白 → 高級感、極簡、永恆

#### B. 字型系統
建立品牌字型規範：
- **標題字型 (Display/Heading Font)**：個性鮮明，傳遞品牌調性
- **內文字型 (Body Font)**：易讀性優先，與標題搭配
- **字重規範**：H1/H2/H3/Body/Caption 各層級大小與字重
- **中英文配對建議**：推薦中文字型（思源黑體、思源宋體等）與英文字型的搭配

#### C. Logo 概念設計
提供文字形式的 Logo 概念說明（由 canvas-design skill 執行視覺產出）：
- **Logo 類型建議**：文字型、圖形型、組合型
- **設計概念**：象徵意義、視覺隱喻、形態描述
- **黑白版本**：說明如何在黑白環境中使用
- **最小使用尺寸**：數位 / 印刷最小尺寸建議

#### D. 品牌視覺風格指南 (Brand Guidelines Document)
整合以上元素，輸出完整的品牌指南文件（.docx 格式，呼叫 docx skill）：

```
品牌指南結構：
1. 品牌故事與核心價值
2. Logo 使用規範（正確/錯誤用法）
3. 色彩系統
4. 字型系統
5. 圖像風格指引
6. 應用範例（名片、信頭、社群封面）
```

---

### Step 3：視覺設計輸出（含個人品牌視覺）

除了品牌識別系統，也負責以下個人品牌視覺規範：

**封面設計**
- YouTube 縮圖視覺規範（字體大小、構圖、情緒表情建議）
- IG 主頁版面規劃（九宮格風格、色調統一原則）

**字卡設計規範**
- 強調字卡：大字 + 高對比色（第一層）
- 說明字卡：小字 + 低調背景（第二層）
- 品牌標語字卡：固定版型 + 品牌色

**版型輸出方式**
- **直接在 Canva 生成** → 使用 Canva MCP（`generate-design-structured`）
- **從 Figma 設計系統讀取** → 使用 Figma MCP（`get_design_context`）
- **需要生成圖像** → 呼叫 `canvas-design` skill
- **需要輸出 Word 文件品牌手冊** → 呼叫 `docx` skill
- **需要輸出簡報版品牌提案** → 呼叫 `pptx` skill
- **需要套用到行銷素材** → 協同 `marketing-materials` skill

## 品牌設計品質標準

好的品牌識別系統應該：

1. **一眼辨識**：在 5 秒內讓目標受眾感知到品牌個性
2. **跨媒體一致**：無論數位或印刷，感覺都像同一個品牌
3. **簡潔耐久**：避免過於複雜或流行元素，能使用 5-10 年
4. **差異化**：在競爭市場中脫穎而出，不與競品混淆
5. **延展性**：在各種尺寸和應用場景都能清晰呈現


```
Step 1 · 確認 Style Token 已就緒
  → visual-style-explorer 已完成（4 象限 + 6 色 + 字體）
  
Step 2 · Logo 概念設計
  → 給 Claude：「我的 Style Token 是 [Q3 暖質感]，
                請給我 5 個 Logo 概念方向（文字型 / 圖形型 / 組合型）」
  → Claude 給概念描述 → 學員選 1 個
  → 用 Canva MCP 或人手做出來（或請設計師執行）

Step 3 · IG 9 宮格規劃
  → 給 Claude：「規劃我未來 30 天 IG 9 宮格」
  → Claude 排「個人故事 / 客戶見證 / 產品教育 / 媽媽日常 / 價值觀」5 內容支柱比例
  → 給每個格子的視覺方向（顏色 / 字體 / 元素）

Step 4 · 限動 + 貼文模板
  → 跟 PART 5.6 T01-T10 限動模板對齊
  → 跟 PART 6 5 貼文封面公式對齊
  
Step 5 · 完整 VI 手冊輸出
  → 用 docx skill 輸出 30 頁 .docx 品牌手冊
  → 內含：Logo 規範 / 色彩 / 字型 / 9 宮格範例 / 應用場景
```

## Bug 排查

| 症狀 | 原因 | 解法 |
|---|---|---|
| Logo 概念太通用 | 沒讀 01 一頁紙的定位句 | 重新讀 01 → 把定位句轉成視覺隱喻 |
| 色彩跟其他帳號撞 | 沒做 Step 1 視覺探索 | 退回 visual-style-explorer 重做 4 象限 |
| 9 宮格沒節奏 | 5 內容支柱比例失衡 | 強制每行（3 格）至少 1 個非產品內容 |
| Canva MCP 用不了 | 沒裝 Canva MCP | 告訴學員：先用文字概念，請外包設計師執行 |
| VI 手冊太長學員看不下 | 塞太多 | 縮成 1 頁版 + 30 頁版兩個（給不同情境） |

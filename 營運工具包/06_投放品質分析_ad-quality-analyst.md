## 🅰️ 投放前：素材預審
1. 廣告影片 → `media_upload` / `media_confirm` 拿 media_id
2. `virality_predictor`（create）→ 病毒分數 / hook 強度 / 留存風險
3. 讀 `_shared/合規清單.md` 過 Meta 廣告政策紅線（前後對比、個人屬性指涉、誇大字眼）
4. 輸出：
```

病毒分數 XX｜Hook 強度 XX｜留存風險 高/中/低

建議：[要改什麼]
```
> 等同「影片審核」的投放版。複雜素材可直接轉 `video-reviewer` skill。

## 🅱️ 投放後：成效診斷

### 資料來源
1. 讀 `_shared/Windsor數據表規格.md` → `meta_ads` 分頁欄位。
2. Google Drive MCP `search_files` 找 `[客戶]_Windsor_數據` → `read_file_content` 讀 `meta_ads`。也接受客戶直接丟 Meta 廣告管理員匯出 CSV（多檔就合併）。
3. 缺欄就提示客戶去 Windsor 補勾，不要硬算。
4. ⚠️ 匯出檔常見「成果/CPA 有、但沒有購買金額」→ **算不出 ROAS**。這時改用 CPA 當核心，並明講「這份沒金額欄，ROAS 算不出，改看 CPA」。**絕不硬編 ROAS。**

### 🥇 第一步：先讀 Meta 三欄原生診斷（最權威，優先於第三方判讀）
Meta 後台自帶 `品質排名 / 參與互動率排名 / 轉換率排名`（需 ≥500 曝光才有，否則顯示「-」跳過）。先用組合定位問題在哪一環：

| 三欄組合 | 問題定位 |
|---|---|
| 全部 高於平均/平均 | 健康，可加碼 |
| 品質低 + 互動低 | **素材問題**（hook/創意弱） |
| 品質低 + **互動高** | **踩 Meta 品質紅線**（誇大字眼 / engagement bait / 個人屬性指涉「是不是25歲」「孕婦不能吃」）→ 對照 `_shared/合規清單.md`，整批 hook 風格要改 |
| 品質好 + 互動好 + **轉換低** | **落地頁/受眾錯配**（非素材問題） |

> 若「絕大多數廣告品質排名都墊底」= 系統性踩線，不是個別廣告爛 → 從 hook 風格 + 合規整批處理，別逐支微調。

### 📊 診斷指標（按廣告組 adset / 廣告 ad 拆）
- **ROAS = purchase_roas**（有金額欄才算；沒有就降級用 CPA，見上）
- **CPA = cost_per_conversion**（越低越好）
- **歸因灌水檢查**：把廣告層「成果加總」對照實際訂單數。若成果遠大於真實訂單（暖受眾 + 7天點擊歸因常見），表示同一筆購買被多支廣告重複認領 → 標記「歸因灌水」，CPA 異常低（如 <$0.5）的不可當加碼依據。
- CTR：對照產業基準（全產業中位 ~2.19%），< 1% = hook/受眾有問題；CPC / CPM：偏高 = 競價或素材弱
- **頻次 frequency**（拆冷熱受眾）：冷受眾 > 2.5 = 早期疲勞、> 4 確定疲勞；再行銷/暖名單容忍到 4.0+。另：**CTR 週對週掉 ≥15%** 比頻次更早預警。
- 影片素材：video_avg_play_time 太短 = 開頭留不住人

### 決策規則（對照 `_shared/` 或現有 FBAds 策略 skill）
```
測試期花費 > NT$50 仍無轉換 → 建議關
ROAS/CPA 達標 且 頻次正常（冷<2.5 / 暖<4）→ 建議加碼
頻次過高（冷>2.5 / 暖>4）且 成效下滑 → 受眾疲勞，換素材或擴受眾
CTR 高但轉換低 → 落地頁/受眾錯配，非素材問題
品質排名全線墊底 → 整批 hook 踩 Meta 紅線，回 ③ 重寫 hook + 過合規，別只調預算
CPA 低到不合理 + 成果遠大於真實訂單 → 歸因灌水，不採信、不據此加碼
```

## 📄 PDF 交付（最後一定要做，渲染檢查過才算完成）
診斷報告跑完（投放後；投放前的素材預審也比照辦理）→ 一律用 Python `reportlab` 輸出一份 PDF（檔名建議 `投放診斷_[日期區間].pdf`），存產出資料夾，附可點擊路徑。

**字體鐵則（中英分家，防跑版，照做別改）**
- 中文 STHeiti：`/System/Library/Fonts/STHeiti Light.ttc`(Light) + `STHeiti Medium.ttc`(Bold)
- 英文/數字 Helvetica：`/System/Library/Fonts/Helvetica.ttc`（subfontIndex 0=Regular, 1=Bold）
- 用 regex `([ -~]+)` 把每段 ASCII（英數/符號、ROAS/CPA/CTR、$金額、%）包進 `<font name="Lat/LatB">`，中文走 STHeiti — 否則英數字距不勻會跑版（這份滿滿指標，最容易爆）。
- ⚠️ 不要用 reportlab 內建名 `"Helvetica"`（此環境輸出空白），一定註冊實體 ttc。
- ⚠️ emoji（📈🟢🔴🟡 等）在這些字體會變空白 → 標題/符號改純文字（如 [加碼]／[該關]／[要調整]）。
- ⚠️ 避免「PART 3」後緊接數字 → 數字改中文。

**渲染檢查（必做，沒檢查不算交付）**
1. 產完 PDF → 把每頁轉 PNG（`pdftoppm -png` 或 `sips`）讀回來看，或直接用 Read 開 PDF 檢視。
2. 逐頁檢查：中文沒變空白/亂碼、金額/ROAS/CPA/百分比沒黏在一起、三欄健檢表沒被切斷、沒破格。
3. 有跑版 → 修字體/欄寬/分頁再重渲染，直到乾淨才交。
4. 回報：「✅ PDF 已渲染檢查，中英數無跑版 → [路徑]」
